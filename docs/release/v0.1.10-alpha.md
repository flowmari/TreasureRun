# TreasureRun v0.1.10-alpha — Repository status and command contract alignment

This alpha release aligns the public repository documentation with the current distribution state and makes the declared plugin command metadata match the active registration paths.

## Changes

- Synchronized the repository status and contributor runtime documentation with the current public alpha state.
- Kept the standard contributor runtime independent of MySQL, with database-backed checks available through the explicit optional mode.
- Made `/treasureReload` the canonical reload command while retaining `/gameReload` as a compatible alias.
- Removed stale metadata for `/game` and `/givespecialemerald`, which had no active registration path.
- Removed the inactive `/dbstatus` registration from the unused `MySQLManager` constructor.
- Removed the redundant case-only `/gameStart` alias.
- Added a command metadata contract test covering command declarations, aliases, permissions, and Java `getCommand(...)` references.
- Synchronized the command reference with the verified plugin metadata.

The documentation synchronization was completed in #137, and the command-contract alignment was completed in #138.
