# v0.1.0-alpha Release Checklist

This checklist prepares TreasureRun for its first alpha release.

## Release goal

Publish TreasureRun as an early alpha portfolio release that demonstrates:

- Minecraft Spigot 1.20.1 mini-game implementation
- multi-layer i18n architecture
- ProtocolLib packet boundary handling
- ResourcePack standard-message coverage
- Fabric runtime language synchronization
- pure-Java PacketI18n JSON localization logic
- CI-backed verification
- Docker-based local validation

## Current engineering claim

TreasureRun separates a Minecraft standard-UI i18n platform boundary into:

- a ProtocolLib boundary adapter
- ResourcePack-backed standard translation assets
- Fabric runtime language synchronization
- a pure-Java packet JSON localizer
- focused unit tests

This allows the project to say:

> TreasureRun separates Minecraft-dependent boundary code from pure packet-localization logic, turning a platform-constrained i18n problem into a testable multi-layer architecture.

## Suggested release title

`v0.1.0-alpha - Platform-boundary i18n architecture preview`

## Suggested release notes

TreasureRun v0.1.0-alpha is an early preview release of a Minecraft Spigot 1.20.1 mini-game plugin focused on maintainable Java architecture and platform-boundary internationalization.

Highlights:

- Spigot / Bukkit mini-game foundation
- ProtocolLib packet-level i18n boundary adapter
- pure-Java PacketI18n JSON localizer
- ResourcePack-based standard Minecraft UI translation assets
- Fabric runtime language synchronization
- 8039-key standard-message ResourcePack coverage for representative language files
- ResourcePack ZIP / SHA verification
- focused tests for PacketI18n boundary separation
- CI-backed validation

This alpha release is intended for portfolio review, technical inspection, and early feedback.
