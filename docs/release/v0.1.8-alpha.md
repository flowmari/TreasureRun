# TreasureRun v0.1.8-alpha — First-run environment and input clarification

This alpha release follows up on external documentation feedback by making the required setup environment and the local connection input more explicit before a first playtest begins.

## Documentation

- Moved “Share first-time feedback” outside the numbered required setup flow and marked it as an optional follow-up.
- Clarified that `localhost:25575` must be entered exactly as shown, including the colon.
- Preserved the verified local Docker address and port instead of introducing a second server-address or port path.
- Made the Bash-compatible-shell requirement explicit before the first setup command.
- Documented WSL 2 with Docker Desktop’s WSL integration as the current Windows setup path.
- Clarified that native PowerShell and Command Prompt are not currently covered by the guide.
- Kept the existing contributor scripts, Docker configuration, and gameplay behavior unchanged.

The follow-ups were tracked in #125 and #127 and implemented in #126 and #128.

## Scope

This release is limited to Player Quickstart clarification and release metadata.

It does not change Java runtime behavior, gameplay logic, configuration behavior, contributor scripts, Docker configuration, i18n sources, language files, ResourcePack assets, database schemas, or ranking implementation.
